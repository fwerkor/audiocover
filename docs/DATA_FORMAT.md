# Data format

## Training input

Select a folder containing `.wav`, `.flac`, `.mp3`, `.m4a`, `.aac`, or `.ogg` files.

Preferred input:

- authorized data
- clean dry files
- stable capture conditions
- no backing music
- no clipping
- 48 kHz WAV when possible

The GUI shows these requirements before training.

## Model package

Generated by:

```bash
audiocover train raw_data models/my_profile --consent
```

Format:

```text
models/my_profile/
  model.yaml
  training_report.json
  dataset/
    report.json
    manifest.jsonl
    wavs/
  simple_timbre.json
  model.pth
  model.index
```

`model.yaml` stores package settings. Normal GUI users do not edit it.

## Render input

One file accepted by FFmpeg. Output is WAV.
